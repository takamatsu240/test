# 議事録フォルダ

このフォルダにWord形式(.docx)またはMarkdown形式(.md)の議事録ファイルを配置すると、GitHub Actionsが自動的にAI解析を実行し、課題とToDoを抽出します。

---

## 📝 使い方

### ステップ1: 議事録ファイルを配置

このフォルダに議事録ファイルをコピーまたは作成します。

**対応形式:**
- Word形式: `.docx`
- Markdown形式: `.md`

**例:**
```
minutes/
├── 2026-02-27_定例会議.docx
├── 2026-03-01_進捗確認.md
└── README.md (このファイル)
```

---

### ステップ2: Gitにコミット&プッシュ

```bash
# ファイルを追加
git add minutes/2026-02-27_定例会議.docx

# コミット
git commit -m "docs: Add meeting minutes 2026-02-27"

# フィーチャーブランチにプッシュ
git push origin feature/your-branch-name
```

**注意:**
- 自動解析は**フィーチャーブランチ**（main/master以外）でのみ実行されます
- mainブランチに直接プッシュしても解析は実行されません

---

### ステップ3: GitHub Actionsの実行を確認

1. GitHubリポジトリの **Actions** タブを開く
2. **Call Minutes Analyzer** ワークフローが実行されていることを確認
3. 完了まで待つ（通常30秒〜1分）

---

### ステップ4: 解析結果を確認

1. タスク管理アプリを開く
2. プロジェクトを選択
3. **「未承認議事録」** セクションに新しい議事録が表示されます
4. 内容を確認して **承認** または **編集** します

---

## 🔄 Word形式の自動変換

Word形式(.docx)の議事録をプッシュすると、以下の処理が自動実行されます：

1. **Word → Markdown変換**
   - Python スクリプトでMarkdown形式に変換
   - 変換後のファイルは `minutes/同じファイル名.md` として保存

2. **自動コミット**
   - 変換されたMarkdownファイルが自動的にコミットされます
   - コミットメッセージ: `docs: Auto-convert minutes from Word to Markdown [skip ci]`

3. **AI解析**
   - 変換後のMarkdownファイルを解析して課題とToDoを抽出

**例:**
```
プッシュ前:
minutes/
└── meeting-2026-02-27.docx

プッシュ後（自動変換）:
minutes/
├── meeting-2026-02-27.docx  ← 元ファイル
└── meeting-2026-02-27.md    ← 自動生成
```

## 📋 議事録のフォーマット

AI解析の精度を高めるため、以下のフォーマットを推奨します：

### 新規課題の記載方法

```markdown
### 課題: タスクのタイトル

**課題内容**: 詳細な説明

**最新状況**: 現在の状況

**対応方針**: 今後の対応方針

**ToDo**: 具体的なタスク
- 担当者: 山田太郎
- 期日: 2026-03-15
- 内容: 実装内容の詳細
```

### 既存課題の更新方法

```markdown
### 課題: タスクのタイトル [既存課題: ISSUE-001]

**最新状況**: 更新内容
```

または

```markdown
**更新対象: ISSUE-001**

**最新状況**: 更新内容
```

### 既存ToDoの更新方法

```markdown
**ToDo**: タスク名 [既存ToDo: TODO-001を更新]

**最新状況**: 進捗状況
```

## ⚙️ カスタマイズ

### フォルダ名を変更したい場合

デフォルトでは `minutes/` フォルダを監視していますが、別の名前に変更できます。

`.github/workflows/call-analyzer.yml` を編集：

```yaml
on:
  push:
    paths:
      - 'docs/meetings/**/*.docx'  # ← カスタムフォルダに変更
      - 'docs/meetings/**/*.md'
```

詳細は `docs/SETUP_ACTIONS.md` を参照してください。

## 🛠️ トラブルシューティング

### ワークフローが起動しない

**確認ポイント:**
- フィーチャーブランチでプッシュしているか？（main/masterブランチでは起動しません）
- コミットメッセージに `[skip ci]` が含まれていないか？
- ファイル拡張子が `.docx` または `.md` か？

### 解析結果が表示されない

**確認ポイント:**
- GitHub Actionsが正常に完了しているか？（Actionsタブで確認）
- Firebase、OpenAI のシークレットが正しく設定されているか？
- 議事録のフォーマットが正しいか？

### Word変換が失敗する

**確認ポイント:**
- Wordファイルが破損していないか？
- Python依存関係（`python-docx`）がインストールされているか？（GitHub Actions側で自動インストール）
